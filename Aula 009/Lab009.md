**1. Instalar kubectl:**
```bash

sudo install -o root -g root -m 0755 kubectl /usr/local/bin/kubectl
kubectl version --client
```

**2. Instalar Minikube:**
```bash
curl -LO https://storage.googleapis.com/minikube/releases/latest/minikube-linux-amd64
sudo install minikube-linux-amd64 /usr/local/bin/minikube
minikube version
```

**3. Configurar Minikube para usar Docker:**
```bash
minikube config set driver docker
```

## Cenário: Definindo uma Aplicação Web Simples (Nginx)

Vamos criar os manifestos YAML necessários para rodar uma aplicação Nginx, garantindo que ela seja escalável e acessível por um endereço estável.

## Seção 1: Preparação do Ambiente e Estrutura

1.  Crie o diretório de trabalho do laboratório.
```bash
mkdir -p ~/aulas_lab/aula010
cd ~/aulas_lab/aula010
```

2.  Crie dois arquivos vazios: um para o Deployment e outro para o Service.
```bash
touch deployment.yaml
touch service.yaml
```

## Seção 2: Criação do Manifesto Deployment (`deployment.yaml`)
O Deployment é o objeto que garante que o estado desejado dos Pods seja mantido (escalabilidade e *self-healing*).
1. Inicialise o VSCode
```bash
code .
```

2. **Copie e cole o seguinte conteúdo no seu arquivo `deployment.yaml`:**

```yaml
# 1. Metadados do Objeto
apiVersion: apps/v1
kind: Deployment
metadata:
  name: web-app-deployment
  labels:
    app: web-app

# 2. Especificação do Estado Desejado
spec:
  # O Deployment deve gerenciar 3 instâncias
  replicas: 3
  
  # Como o Deployment encontra os Pods que ele deve gerenciar
  selector:
    matchLabels:
      app: web-app
      
  # Template dos Pods (O QUE DEVE SER RODADO)
  template:
    metadata:
      # As labels são usadas pelo Service para encontrar este Pod
      labels:
        app: web-app
    spec:
      containers:
      - name: nginx-container
        image: nginx:alpine
        ports:
        - containerPort: 80
```

## Seção 3: Criação do Manifesto Service (`service.yaml`)

O Service é o *Load Balancer* interno e o endereço estável para o Deployment. Vamos usar o tipo `NodePort` para simular o acesso externo.

1. **Copie e cole o seguinte conteúdo no seu arquivo `service.yaml`:**

```yaml
# 1. Metadados do Objeto
apiVersion: v1
kind: Service
metadata:
  name: web-app-service

# 2. Especificação do Estado Desejado
spec:
  # Tipo NodePort: Expõe o serviço na porta de todos os Worker Nodes
  type: NodePort
  
  # Selector: O Service encontra os Pods usando esta label
  selector:
    app: web-app
    
  # Mapeamento de Portas
  ports:
    # service.port: Porta onde o Service escuta (interna ao cluster)
    - port: 80
      # targetPort: Porta que o contêiner expõe (porta 80 do Nginx)
      targetPort: 80
      # nodePort: Porta que será aberta em cada Node (geralmente entre 30000-32767)
      nodePort: 30080
```

## Seção 4: Validação (Conceitual)

Responda às seguintes perguntas com base nos manifestos criados:

1.  **Escalabilidade:** Se um Pod morrer, qual objeto no K8s é acionado para criar um novo Pod e restaurar a contagem para 3?
2.  **Conexão Interna:** Se um segundo Pod precisar se comunicar com esta aplicação, qual objeto ele deve usar? (O Service ou o Deployment?)
3.  **Fluxo de Roteamento:** Quando o tráfego externo chega na porta `30080` de um Node, ele é roteado primeiramente para o Service e depois para qual porta do contêiner?
4.  **Label de Conexão:** Qual é a *label* que conecta o `web-app-deployment` ao `web-app-service`?

## Seção 5: Deploy em Ambiente Minikube

Agora vamos aplicar os manifestos em um cluster Kubernetes local usando Minikube.

### 5.1: Inicializar Minikube

1. **Iniciar o cluster Minikube:**
```bash
minikube start
```

2. **Verificar status do cluster:**
```bash
kubectl cluster-info
kubectl get nodes
```

### 5.2: Aplicar os Manifestos

1. **Aplicar o Deployment:**
```bash
kubectl apply -f deployment.yaml
```

2. **Aplicar o Service:**
```bash
kubectl apply -f service.yaml
```

3. **Verificar recursos criados:**
```bash
kubectl get deployments
kubectl get pods
kubectl get services
```

### 5.3: Testar a Aplicação

1. **Obter URL do serviço:**
```bash
minikube service web-app-service --url
```

### 5.4: Escalar o Deployment

1. **Escalar para 5 réplicas:**
```bash
kubectl scale deployment web-app-deployment --replicas=5
```

2. **Verificar escala:**
```bash
kubectl get pods
```

### 5.5: Limpeza do Ambiente

1. **Remover recursos:**
```bash
kubectl delete -f service.yaml
kubectl delete -f deployment.yaml
```

2. **Parar Minikube:**
```bash
minikube stop
```

3. Apagar pastas
```bash
cd ~
rm -rf ~/aulas_lab/aula010  
```
